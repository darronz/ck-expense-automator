<objective>
Set up the two-entrypoint world split and pure utility layer for the panel.

Purpose: The panel requires an isolated-world content script (ck-panel.content.ts) because createShadowRootUi calls browser.runtime.getURL() which is unavailable in MAIN world. The utility module (panel-utils.ts) provides all pure functions needed for rendering — foreign currency extraction, VAT divisibility check, category sorting, and match pattern derivation — with full unit test coverage before the panel DOM is written.

Output: Two content script entrypoints wired together via custom DOM event, panel-utils.ts with tests, panel.css scaffold, and empty panel.test.ts for Wave 2.

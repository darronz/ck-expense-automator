<objective>
Implement the unmatched items section: inline assignment form with category dropdown, vendor pre-fill, VAT fields, VAT divisibility hint, save-as-rule flow with editable pattern, and client-side validation.

Purpose: Unmatched items are the user's path to handling novel expenses. The inline form must be frictionless — smart defaults (vendor extracted, category pre-selected, save-as-rule checked) mean the user typically changes only 1-2 fields before submitting.

Output: Completed unmatched section in ui/panel.ts with inline form, updated panel.css for form styles, and unit tests for the new exported functions.
</objective>

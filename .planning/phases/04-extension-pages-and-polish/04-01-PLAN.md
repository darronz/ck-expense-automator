WXT entrypoint pattern (HTML page):
- File at entrypoints/popup/index.html — standard HTML with <script type="module" src="./main.ts">
- File at entrypoints/popup/main.ts — plain TypeScript, no defineContentScript wrapper needed
- WXT automatically picks up entrypoints/popup/ as the popup and entrypoints/options/ as the options page
- Use browser.* (WXT global) for extension APIs in entrypoints, not chrome.*

Message protocol (new — to be established by this plan):
- Popup → content script: chrome.tabs.sendMessage(tabId, { type: 'CK_TOGGLE_PANEL' })
- Content script → popup (response): { matched: number, unmatched: number }
- Content script must handle CK_TOGGLE_PANEL in its message listener

**entrypoints/popup/main.ts** — Popup logic:

1. On DOMContentLoaded, query the active tab with `browser.tabs.query({ active: true, currentWindow: true })`.
2. Check if `tab.url` matches `https://portal.churchill-knight.co.uk/ExpenseItems/*` using a simple string startsWith check.
3. If NOT on CK page: render `<p class="status-inactive">Navigate to CK Expenses to use</p>` in #app.
4. If ON CK page:
   a. Send message `{ type: 'CK_GET_STATE' }` to the tab via `browser.tabs.sendMessage(tab.id!, { type: 'CK_GET_STATE' })`. Wrap in try/catch — if content script is not ready, show "Panel loading..." message.
   b. Response type: `{ matched: number; unmatched: number; panelVisible: boolean }`.
   c. Render in #app:
      ```html
      <div class="status-active">
        <div class="counts">
          <div class="count-item">
            <span class="count-matched">{matched}</span>
            <span>Matched</span>
          </div>
          <div class="count-item">
            <span class="count-unmatched">{unmatched}</span>
            <span>Unmatched</span>
          </div>
        </div>
        <div class="actions">
          <button id="btn-toggle" class="btn-primary {panelVisible ? 'btn-toggle-active' : ''}">
            {panelVisible ? 'Hide Panel' : 'Show Panel'}
          </button>
          <button id="btn-options" class="btn-secondary">Options</button>
        </div>
      </div>
      ```
   d. "Show/Hide Panel" button: sends `{ type: 'CK_TOGGLE_PANEL' }` to tab, then closes popup with `window.close()`.
   e. "Options" button: calls `browser.runtime.openOptionsPage()` then `window.close()`.

5. Also add the message handler to **entrypoints/ck-panel.content.ts** (read that file first, add to its main() function):
   ```typescript
   browser.runtime.onMessage.addListener((message, _sender, sendResponse) => {
     if (message.type === 'CK_GET_STATE') {
       sendResponse({
         matched: currentMatchResult?.matched.length ?? 0,
         unmatched: currentMatchResult?.unmatched.length ?? 0,
         panelVisible: isPanelVisible,
       });
       return true;
     }
     if (message.type === 'CK_TOGGLE_PANEL') {
       togglePanelVisibility();
       return true;
     }
   });
   ```
   Where `currentMatchResult` and `isPanelVisible` are module-level variables. Read ck-panel.content.ts carefully to understand what state variables exist and add the listener in the appropriate place. If `currentMatchResult` and `isPanelVisible` are not yet module-level, promote them. `togglePanelVisibility()` should show/hide the `shadowHost` element.

**Page initialisation (DOMContentLoaded):**
1. Load rules with `getRules()`.
2. Render the full page into `#app`:
   - `<h1>CK Expense Automator — Rule Manager</h1>`
   - Toolbar with rule count ("N rules") on the left and two buttons on the right: "Add Rule" and "Reset to Defaults"
   - Rules table (see columns below)
3. Attach event listeners.

**Table columns:**
1. **Enabled** — checkbox (native `<input type="checkbox">`, checked = rule.enabled). On change: call `updateRule({ ...rule, enabled: !rule.enabled })` and re-render the row's enabled badge.
2. **Name** — rule.name (plain text in view mode, `<input>` in edit mode)
3. **Match Pattern** — rule.matchPattern (plain text in view, `<input>` in edit). Show as `<code>` element in view mode.
4. **Category** — `getCategoryLabel(rule.nominalId)` in view mode; `<select>` with all CATEGORIES options in edit mode. Selected option = rule.nominalId.
5. **Vendor** — rule.purchasedFrom (plain text in view, `<input>` in edit)
6. **VAT** — Show "£X.XX" if hasVat && vatAmount, "X%" if hasVat && vatPercentage, "None" if !hasVat. In edit mode show: hasVat checkbox, vatAmount input (number), vatPercentage input (number). Only one of vatAmount/vatPercentage should be non-null; if user fills vatAmount, clear vatPercentage and vice versa.
7. **Usage** — `rule.matchCount ?? 0` uses, last used = `rule.lastUsed ? new Date(rule.lastUsed).toLocaleDateString('en-GB') : 'Never'`. Both as `.muted` text. Read-only (never editable).
8. **Actions** — In view mode: [Edit] [Delete] buttons. In edit mode: [Save] [Cancel] buttons.

**Inline edit behaviour:**
- Only one row editable at a time. Clicking Edit on a second row while one is open cancels the first.
- Edit mode: replace all editable `<td>` contents with input/select elements pre-filled with current values.
- Save: call `updateRule(editedRule)` where editedRule is constructed from the input values. Re-render the row in view mode.
- Cancel: re-render the row in view mode without saving.
- Validation before Save: name and matchPattern must not be empty. If empty, add a red border to the invalid input and abort.

**Delete behaviour:**
- Call `window.confirm('Delete rule "' + rule.name + '"?')`. If confirmed, call `deleteRule(rule.id)` then remove the row from the table and decrement the count in the toolbar.

**Reset to Defaults button:**
- Call `window.confirm('Reset ALL rules to defaults? This will delete all your custom rules.')`. If confirmed, call `saveRules(DEFAULT_RULES)` then re-render the entire page (call the init function again).

**Add Rule form (shown when "Add Rule" is clicked):**
- Show a form above the table (`.add-form` div, hidden by default, shown on click).
- Fields: Name (text), Match Pattern (text), Category (select from CATEGORIES), Description (text), Vendor (text), Has VAT (checkbox), VAT Amount (number, shown when Has VAT checked), VAT % (number, shown when Has VAT checked, mutually exclusive with VAT Amount).
- Submit: construct a new ExpenseRule with `crypto.randomUUID()` as id, `enabled: true`, `createdAt: new Date().toISOString()`, `matchFlags: 'i'`, `matchCount: 0`, `lastUsed: undefined`. Call `addRule(newRule)` then re-render the page.
- Cancel: hide the form without saving.

**chrome.storage.onChanged listener** (for live sync from panel changes):
```typescript
browser.storage.onChanged.addListener((changes, area) => {
  if (area === 'sync' && changes['rules']) {
    // Re-render page to reflect updated rules
    initPage();
  }
});
```
This ensures the options page stays in sync if rules are saved from the content script (e.g. "save as rule" from unmatched item flow).

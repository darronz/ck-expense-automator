Response parsing logic (from 02-RESEARCH.md Pattern 5):
- If `response.redirected === true`:
  - URL contains '/login' or '/account/' (case-insensitive) OR does not start with 'https://portal.churchill-knight.co.uk/' → SESSION_EXPIRED
  - Otherwise → success (PRG pattern)
- If not redirected and HTTP 200:
  - Parse body HTML with DOMParser, query `.validation-summary-errors li` and `.field-validation-error`
  - If errors found → VALIDATION_ERROR
  - Otherwise → success

ASP.NET validation HTML structure (from 02-RESEARCH.md):
```html
<div class="validation-summary-errors">
  <ul><li>The VAT amount is too high.</li></ul>
</div>
<span class="field-validation-error">The Description field is required.</span>
```
